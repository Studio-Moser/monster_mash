Game Prototypes - Evolution of Mechanics

Prototype 1

Basic Color Loop

Board: 5x5 grid (25 spaces)
Goal: Delay board filling, score points
Scoring: +1 point per eat
Spawn: ~70% characters / 30% actions
🎲 Characters: 🔴 Red, 🔵 Blue, 🟢 Green, 🟡 Yellow
Food Chain: Red→Blue→Green→Yellow→Red (circular)
Direction: Each faces ↑↓←→
Actions: ↻↺ (rotate), ↑→↓← (move one space)

The Original: Established core concept of directional eating with circular food chain. Characters stay their type forever (no evolution). Simple +1 scoring per eat.

Limitations: No animations - pieces just appear/disappear instantly. No visual feedback showing which pieces are being evaluated. Difficult to track cascade logic. No hover preview.

Key Innovation: Proved circular loop concept works - pieces never reach "final form", keeping all active.

Prototype 3

Animal Food Chain + Animations

Board: 5x5 grid (25 spaces)
Goal: Chain reactions and score
Scoring: +1 point per eat
Spawn: ~70% characters / 30% actions
🎲 Characters: 🧑 Human, 🐟 Fish, 🐸 Frog, 🦟 Mosquito
Food Chain: Human→Fish→Frog→Mosquito→Human (thematic loop!)
Size Hierarchy: Human (32px) > Fish (26px) > Frog (20px) > Mosquito (14px)
Actions: ↻↺ (rotate), ↑→↓← (move)

Major UX Upgrade: First prototype with smooth animations! Pieces slide from cell to cell (300ms transitions) instead of instant teleport. Can actually SEE the eating happen.

Visual Feedback Added:
• Yellow glow shows which piece is being evaluated
• Green double-flash when match is found
• Hover preview shows semi-transparent next piece
• Clockwise neighbor checking visible (125ms per check)

Thematic Identity: Replaced abstract colored circles with relatable animal food chain. Size differences make "who eats whom" instantly clear. Mosquito killing human adds thematic twist.

Learning: Visual feedback is CRITICAL. Complex cascade logic becomes understandable when you can see it working step-by-step.

Prototype 4

Size-Based Hierarchy

Board: 5x5 grid (25 spaces)
Goal: Chain reactions with size progression
Scoring: +1 point per eat
Spawn: ~85% characters / 15% actions
🎲 Characters: 🟡 Yellow, 🟠 Orange, 🔴 Red, 🟣 Purple, 🔵 Blue
Food Chain: Yellow→Orange→Red→Purple→Blue (LINEAR progression)
Size Hierarchy: Yellow (10px) → Orange (18px) → Red (28px) → Purple (40px) → Blue (50px)
Actions: ↻↺ (rotate), ↑→↓← (move), 💀 (SKULL - remove any piece)
Visual Polish: All animations from P3

Experiment: Can size alone communicate eating order without themed emojis? YES - dramatic size differences (10px → 50px) make hierarchy obvious.

Linear vs Circular: Unlike P1-3's loop, this is one-way progression. Blue is "final form" - can't be eaten, becomes permanent obstacle.

Skull Action: Emergency tool to remove problematic pieces (especially stuck blue dots). Makes game easier but adds tactical option.

Problem Discovered: End-state pieces (blue) fill board with dead spaces. Reduces strategic options. Linear progression less interesting than perpetual loop.

Prototype 6

Color Mixing Evolution

Board: 5x5 grid (25 spaces)
Goal: Mix colors to evolve toward black
Scoring: +1 point per merge
Evolution: 5 levels before black (final form)
🎲 Starting Colors: 🔴 Red, 🔵 Blue, 🟡 Yellow (primaries only)
Mixing Rule: Any color can eat any other color of SAME LEVEL
Actions: ↻↺ (rotate), ↑→↓← (move), 💀 (remove)
Visual Polish: Colored circles with black outlines, all P3 animations

True Paint Mixing: Uses RYB color space conversion for realistic paint-style blending:
• Red + Blue = Purple (not grey!)
• Blue + Yellow = Green
• Yellow + Red = Orange
• Colors progressively muddy toward black

Evolution System:
Level 0 (16px) → Level 1 (22px) → Level 2 (28px) → Level 3 (36px) → Level 4 (44px) → Level 5 Black (52px)

Critical Flaw: Same-level merging = 2048 clone. Only 5 eating opportunities before pieces become black (dead). Limited chain potential. Beautiful visuals but derivative gameplay.

Verdict: Too similar to existing merge-puzzle games. Lacks unique identity despite pretty color mixing.

Prototype 7

Chain Reaction Master

Board: 4x4 grid (16 spaces)
Goal: Create massive chain reactions and maximize score
Scoring: Chain multiplier - n² points (1=1, 2=4, 3=9, 4=16 pts!)
Difficulty: 90% character spawn / 10% actions
🎲 Characters: 🧑 Human, 🐟 Fish, 🐸 Frog, 🦟 Mosquito
Food Chain: Human→Fish→Frog→Mosquito→Human (circular loop, no evolution)
Size hierarchy shows eating order visually
Direction: Each faces ↑↓←→ (randomly assigned)

🎯 Action Pieces (10% spawn rate):
• ↻↺ Rotate in place
• ↑→↓← Move + rotate to face that direction

⚡ Multi-Eat: Pieces eat ALL valid targets in their path until blocked. Example: Human→ facing Fish-Fish-Fish eats all 3!

🌪️ Direction Inheritance: After eating, pieces inherit victim's facing direction. Creates unpredictable zigzag cascades:
• Human↑ eats Fish← → becomes Human←
• Human← eats Frog↓ → becomes Human↓
• Chains veer in unexpected directions like Bejeweled

💯 Chain Multiplier: Counts ALL eats across entire cascade (including neighbor-triggered). 3+ chains trigger confetti celebration!

🎯 Intelligent Logic: Last moved piece checks first → clockwise neighbors (top→right→bottom→left) → first valid eater becomes focus → repeats. Yellow glow shows evaluation, green flash shows matches.

Why it's unique: NOT like 2048 (no merging/evolution), NOT like Bejeweled (manual placement + directional facing). Direction inheritance creates emergent cascades that feel "lucky" and shareable!
▶ Play Prototype 7

Prototype Evolution: From basic color loops → size hierarchies → color mixing → chain reaction mastery

Key Discovery: Circular food chain (no evolution) + multi-eat + direction inheritance = unique, non-derivative gameplay
